\section{Programming Paradigms} 
The logical approach in software engineering is descriptive of how programming language is implemented.

\subsection{Reasons for studying programming paradigms}
\begin{itemize}
    \item \textbf{Increased ability to express ideas}
    \item \textbf{Improved background for choose appropriate language}
    \item \textbf{Increased ability for learn new languages}
\end{itemize}

\section{Programming Domains} 
\begin{itemize}
    \item Scientific Computing :  Fortran
    \item Business Applications : COBOL
    \item AI : LISP
    \item Systems Programming : C/C++
    \item Web Software : Markup languages (HTML/CSS) , Scripting languages (JavaScript , PHP) or General-Purpose languages (Java , Python)
\end{itemize}


\section{Language Evalutation Criteria}

\begin{itemize}
    \item \textbf{Readability} : Programs can read and understood
    \item \textbf{Writability} : Language can be used to create programs
    \item \textbf{Reliability} : Conformance to specifications
    \item \textbf{Cost} : The ultimate total cost
\end{itemize}

\subsection{Evaluation Criteria: Readability}
\begin{itemize}
    \item \textbf{Overall Simplicity} : manageable set of features and constructs
    \item \textbf{Orthogonality}
    \item \textbf{Data types} : Adequate predefined data types

    \item \textbf{Simplicity and Orthogonality}
